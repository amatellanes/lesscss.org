<h2>1.4.0</h2>

<p>We have released 1.4.0. This includes new features such as extends, the data-uri function and more maths functions. See the <a href="https://github.com/cloudhead/less.js/blob/master/CHANGELOG.md">changelog</a> for a full list of changes.</p>

<p>There are some known <span class="warning">breaking changes</span>.</p>

<p><code>@import-once</code> is removed and is now default behaviour for <code>@import</code>.</p>

<p><code>(~".myclass_@{index}") { ...</code> selector interpolation is deprecated, do this instead <code>.myclass_@{index} { ...</code>. This works in 1.3.1 onwards.</p>

<p>The browser version no longer bundles a version of es5-shim.js - the version we previously used was inaccurate and the new version is significantly larger. Please include your choice of es-5 shim or only use on modern browsers.</p>

<p>We have introduced a optional strictMath mode, where math is required to be in parenthesis, e.g.</p>

<pre><code>(1 + 1) // 2
1 + 1 // 1+1
</code></pre>

<p>In 1.4.0 this option is turned off, but we intend to turn this on by default. We recommend you upgrade code and switch on the option (&ndash;strict-math=on in the command line or strictMath: true in JavaScript). Code written with brackets is backwards compatible with older versions of the less compiler.</p>

<p>We also added a strict units option (strictUnits: true or strict-units=on) and this causes lessc to validate the units used are valid (e.g. 4px/2px = 2, not 2px and 4em/2px throws an error). There are no longer any plans to switch this option on permanently, but some users will find it useful for bug finding.</p>

<p>Unit maths is done, so <code>(4px * 3em) / 4px</code> used to equal <code>3px</code> and it now equals <code>3em</code>. However we do not cancel units down to unitless numbers unless strict units is on.
The selector interpolation, maths and units changes can be made to your less now and will compile fine with less 1.3.3.</p>

<h2>Variables</h2>

<p>Variables allow you to specify widely used values in a single place, and then re-use them throughout the style sheet,
making global changes as easy as changing one line of code.</p>

<table class="code-example" cellpadding="0">
<tr><td>
<pre class="less-example">
<code>// LESS

@color: #4D926F;

#header {
color: @color;
}
h2 {
color: @color;
}</code></pre>
</td><td>
<pre class="css-output"><code>/* Compiled CSS */

#header {
color: #4D926F;
}
h2 {
color: #4D926F;
}</code></pre></td>
</tr>
</table>


<h2>Mixins</h2>

<p>Mixins allow you to embed all the properties of a class into another class by
simply including the class name as one of its properties. It&rsquo;s just like variables,
but for whole classes. Mixins can also behave like functions, and take arguments,
as seen in the example below.</p>

<table class="code-example" cellpadding="0">
<tr><td>
<pre class="less-example"><code>// LESS

.rounded-corners (@radius: 5px) {
-webkit-border-radius: @radius;
-moz-border-radius: @radius;
-ms-border-radius: @radius;
-o-border-radius: @radius;
border-radius: @radius;
}

#header {
.rounded-corners;
}
#footer {
.rounded-corners(10px);
}</code></pre></td>

<td>
<pre class="css-output"><code>/* Compiled CSS */

#header {
-webkit-border-radius: 5px;
-moz-border-radius: 5px;
-ms-border-radius: 5px;
-o-border-radius: 5px;
border-radius: 5px;
}
#footer {
-webkit-border-radius: 10px;
-moz-border-radius: 10px;
-ms-border-radius: 10px;
-o-border-radius: 10px;
border-radius: 10px;
}</code></pre>
</td></tr>
</table>


<h2>Nested Rules</h2>

<p>Rather than constructing long selector names to specify inheritance,
in Less you can simply nest selectors inside other selectors.
This makes inheritance clear and style sheets shorter.</p>

<table class="code-example" cellpadding="0">
<tr><td>
<pre class="less-example">
<code>// LESS

#header {
h1 {
font-size: 26px;
font-weight: bold;

&:hover { border-width: 1px }

}
}

</code></pre></td>

</code></pre>

<h2>Functions &amp; Operations</h2>

<p>Are some elements in your style sheet proportional to other elements?
Operations let you add, subtract, divide and multiply property values and colors,
giving you the power to create complex relationships between properties. Operations
should only be performed within parentheses in order to ensure compatibility with CSS.
Functions map one-to-one with JavaScript code, allowing you to manipulate values however
you want.</p>

<table class="code-example" cellpadding="0">
<tr><td>
<pre class="less-example">
<code>// LESS

@the-border: 1px;
@base-color: #111;
@red: #842210;

#header {
color: (@base-color * 3);
border-left: @the-border;
border-right: (@the-border * 2);
}
#footer {
color: (@base-color + #003300);
border-color: desaturate(@red, 10%);
}

</code></pre></td>

<td>
<pre class="css-output"><code>/* Compiled CSS */

#header {
color: #333;
border-left: 1px;
border-right: 2px;
}
#footer {
color: #114411;
border-color: #7d2717;
}

</code></pre>
</td></tr>

<h1>Client-side usage</h1>

<p>Client-side is the easiest way to get started and good for developing your LESS. For production and especially
if performance is important, we recommend pre-compiling using node or one of the many third party tools.</p>

<p>Link your <code>.less</code> stylesheets with the <code>rel</code> set to &ldquo;<code>stylesheet/less</code>&rdquo;:</p>

<pre><code>&lt;link rel="stylesheet/less" type="text/css" href="styles.less" /&gt;
</code></pre>

<p>Then download <code>less.js</code> from the top of the page, and include it in the <code>&lt;head&gt;</code> element of your page, like so:</p>

<pre><code>&lt;script src="less.js" type="text/javascript"&gt;&lt;/script&gt;
</code></pre>

<p>Make sure you include your stylesheets <em>before</em> the script.</p>

<p>You can set options by setting things on a global LESS object before the script. E.g.</p>

<pre><code>&lt;script type="text/javascript"&gt;
less = {
env: "development", // or "production"
async: false, // load imports async
fileAsync: false, // load imports async when in a page under
// a file protocol
poll: 1000, // when in watch mode, time in ms between polls
functions: {}, // user functions, keyed by name
dumpLineNumbers: "comments", // or "mediaQuery" or "all"
relativeUrls: false,// whether to adjust url's to be relative
// if false, url's are already relative to the
// entry less file
rootpath: ":/a.com/"// a path to add on to the start of every url
//resource
};
&lt;/script&gt;
&lt;script src="less.js" type="text/javascript"&gt;&lt;/script&gt;
</code></pre>

<h2>Watch mode</h2>

<p><em>Watch mode</em> is a client-side feature which enables your styles to refresh automatically as they are changed.</p>

<p>To enable it, append &lsquo;<code>#!watch</code>&rsquo; to the browser URL, then refresh the page. Alternatively, you can
run <code>less.watch()</code> from the console.</p>

<h2>Modify variables</h2>

<p><em>modifyVars</em> enables modification of LESS variables in run-time. When called with new values, the LESS file
is recompiled without reloading. Simple basic usage:</p>

<pre><code>less.modifyVars({
'@buttonFace': '#5B83AD',
'@buttonText': '#D9EEF2'
});
</code></pre>

<h2>Debugging</h2>

<p>It is possible to output rules in your CSS which allow tools to locate the source of the rule.</p>

<p>Either specify the option <code>dumpLineNumbers</code> as above or add <code>!dumpLineNumbers:mediaQuery</code> to the url.</p>

<p>You can use the &ldquo;comments&rdquo; option with <a href="https://addons.mozilla.org/en-us/firefox/addon/fireless/">FireLESS</a> and
the &ldquo;mediaQuery&rdquo; option with FireBug/Chrome dev tools (it is identical to the SCSS media query debugging format).</p>

<h1>Server-side usage</h1>

<h2>Installation</h2>

<p>The easiest way to install LESS on the server, is via <a href="http://github.com/isaacs/npm">npm</a>, the node package manager, as so:</p>

<pre><code>$ npm install -g less
</code></pre>

<h2>Command-line usage</h2>

<p>Once installed, you can invoke the compiler from the command-line, as such:</p>

<pre><code>$ lessc styles.less
</code></pre>

<p>This will output the compiled CSS to <code>stdout</code>, you may then redirect it to a file of your choice:</p>

<pre><code>$ lessc styles.less &gt; styles.css
</code></pre>

<p>To output minified CSS, simply pass the <code>-x</code> option. If you would like more involved minification,
the <a href="http://developer.yahoo.com/yui/compressor/css.html">YUI CSS Compressor</a> is also available with
the <code>--yui-compress</code> option.</p>

<p>To see all the command line options run lessc without parameters.</p>

<h2>Usage in Code</h2>

<p>You can invoke the compiler from node, as such:</p>

<pre><code>var less = require('less');

less.render('.class { width: (1 + 1) }', function (e, css) {
console.log(css);
});
</code></pre>

<p>which will output</p>

<pre><code>.class {
width: 2;
}
</code></pre>

<p>you may also manually invoke the parser and compiler:</p>

<pre><code>var parser = new(less.Parser);

parser.parse('.class { width: (1 + 1) }', function (err, tree) {
if (err) { return console.error(err) }
console.log(tree.toCSS());
});
</code></pre>

<h2>Configuration</h2>

<p>You may pass some options to the compiler:</p>

<pre><code>var parser = new(less.Parser)({
paths: ['.', './lib'], // Specify search paths for @import directives
filename: 'style.less' // Specify a filename, for better error messages
});

parser.parse('.class { width: (1 + 1) }', function (e, tree) {
tree.toCSS({ compress: true }); // Minify CSS output
});
</code></pre>

<h1>The Language</h1>
<div class="content">
<p>As an extension to CSS, LESS is not only backwards compatible with CSS, but the extra features it adds use <em>existing</em> CSS syntax. This makes learning LESS a <em>breeze</em>, and if in doubt, lets you fall back to CSS.</p>

<h2>Variables</h2>

<p>These are pretty self-explanatory:</p>

<pre><code>@nice-blue: #5B83AD;
@light-blue: (@nice-blue + #111);

#header { color: @light-blue; }
</code></pre>

<p>Outputs:</p>

<pre><code>#header { color: #6c94be; }
</code></pre>

<p>It is also possible to define variables with a variable name:</p>

<pre><code>@fnord: "I am fnord.";
@var: 'fnord';
content: @@var;
</code></pre>

<p>Which compiles to:</p>

<pre><code>content: "I am fnord.";
</code></pre>

<p>When defining a variable twice, the last definition of the variable is used, searching from the current scope upwards. This is similar to css itself where the last property inside a definition is used to determine the value.</p>

<p>For instance:</p>

<pre><code>@var: 0;
.class1 {
@var: 1;
.class {
@var: 2;
three: @var;
@var: 3;
}
one: @var;
}
</code></pre>

<p>Compiles to:</p>

<pre><code>.class1 .class {
three: 3;
}
.class {
one: 1;
}
</code></pre>

<p>Variables are lazy loaded and do not have to be declared before being used.</p>

<p>Valid less snippet:</p>

<pre><code>lazy-eval {
width: @var;
}

@var: @a;
@a: 9%;
</code></pre>

<p>this is valid less too:</p>

<pre><code>.lazy-eval-scope {
width: @var;
@a: 9%;
}

@var: @a;
@a: 100%;
</code></pre>

<p>both compile into:</p>

<pre><code>.lazy-eval-scope {
width: 9%;
}
</code></pre>

<h2>Mixins</h2>

<p>In LESS, it is possible to include a bunch of properties from one ruleset into another ruleset. So say we have the following class:</p>

<pre><code>.bordered {
border-top: dotted 1px black;
border-bottom: solid 2px black;
}
</code></pre>

<p>And we want to use these properties inside other rulesets. Well, we just have to drop in the name of
the class in any ruleset we want to include its properties, like so:</p>

<pre><code>#menu a {
color: #111;
.bordered;
}
.post a {
color: red;
.bordered;
}
</code></pre>

<p>The properties of the <code>.bordered</code> class will now appear in both <code>#menu a</code> and <code>.post a</code>:</p>

<pre><code>#menu a {
color: #111;
border-top: dotted 1px black;
border-bottom: solid 2px black;
}
.post a {
color: red;
border-top: dotted 1px black;
border-bottom: solid 2px black;
}
</code></pre>

<p>Any CSS <em>class</em> or <em>id</em> ruleset can be mixed-in that way.</p>

<p>Note: Variables are also mixed in, so variables from a mixin will be placed into the current scope. This is contentious and may change in the future.</p>

<h2>Parametric Mixins</h2>

<p>LESS has a special type of ruleset which can be mixed in like classes, but accepts parameters. Here&rsquo;s the canonical example:</p>

<pre><code>.border-radius (@radius) {
border-radius: @radius;
-moz-border-radius: @radius;
-webkit-border-radius: @radius;
}
</code></pre>

<p>And here&rsquo;s how we can mix it into various rulesets:</p>

<pre><code>#header {
.border-radius(4px);
}
.button {
.border-radius(6px);
}
</code></pre>

<p>Parametric mixins can also have default values for their parameters:</p>

<pre><code>.border-radius (@radius: 5px) {
border-radius: @radius;
-moz-border-radius: @radius;
-webkit-border-radius: @radius;
}
</code></pre>

<p>We can invoke it like this now:</p>

<pre><code>#header {
.border-radius;
}
</code></pre>

<p>And it will include a 5px border-radius.</p>

<p>You can also use parametric mixins which don&rsquo;t take parameters. This is useful if you want to hide the ruleset from the CSS output,
but want to include its properties in other rulesets:</p>

<pre><code>.wrap () {
text-wrap: wrap;
white-space: pre-wrap;
white-space: -moz-pre-wrap;
word-wrap: break-word;
}

pre { .wrap }
</code></pre>

<p>Which would output:</p>

<pre><code>pre {
text-wrap: wrap;
white-space: pre-wrap;
white-space: -moz-pre-wrap;
word-wrap: break-word;
}
</code></pre>

<h3>Mixins With Multiple Parameters</h3>

<p>Parameters are either <em>semicolon</em> or <em>comma</em> separated. It is recommended to use <em>semicolon</em>. The symbol comma has double meaning: it can be interpreted either as a mixin parameters separator or css list separator.</p>

<p>Using comma as mixin separator makes it impossible to create comma separated lists as an argument. On the other hand, if the compiler sees at least one semicolon inside mixin call or declaration, it assumes that arguments are separated by semicolons and all commas belong to css lists:</p>

<ul>
<li>two arguments and each contains comma separated list: <code>.name(1, 2, 3; something, else)</code>,</li>
<li>three arguments and each contains one number: <code>.name(1, 2, 3)</code>,</li>
<li>use dummy semicolon to create mixin call with one argument containing comma separated css list: <code>.name(1, 2, 3;)</code>,</li>
<li>comma separated default value: <code>.name(@param1: red, blue;)</code>.</li>
</ul>


<p>It is legal to define multiple mixins with the same name and number of parameters. Less will use properties of all that can apply. If you used the mixin with one parameter e.g. <code>.mixin(green);</code>, then properties of all mixins with exactly one mandatory parameter will be used:</p>

<pre><code>.mixin(@color) {
color-1: @color;
}
.mixin(@color; @padding:2) {
color-2: @color;
padding-2: @padding;
}
.mixin(@color; @padding; @margin: 2) {
color-3: @color;
padding-3: @padding;
margin: @margin @margin @margin @margin;
}
.some .selector div {
.mixin(#008000);
}
</code></pre>

<p>compiles into:</p>

<pre><code>.some .selector div {
color-1: #008000;
color-2: #008000;
padding-2: 2;
}
</code></pre>

<h3>The <code>@arguments</code> variable</h3>

<p><code>@arguments</code> has a special meaning inside mixins, it contains all the arguments passed, when the mixin was called. This is useful
if you don&rsquo;t want to deal with individual parameters:</p>

<pre><code>.box-shadow (@x: 0; @y: 0; @blur: 1px; @color: #000) {
box-shadow: @arguments;
-moz-box-shadow: @arguments;
-webkit-box-shadow: @arguments;
}
.box-shadow(2px; 5px);
</code></pre>

<p>Which results in:</p>

<pre><code> box-shadow: 2px 5px 1px #000;
-moz-box-shadow: 2px 5px 1px #000;
-webkit-box-shadow: 2px 5px 1px #000;
</code></pre>

<h3>Advanced arguments and the <code>@rest</code> variable</h3>

<p>You can use <code>...</code> if you want your mixin to take a variable number of arguments. Using this after a variable name will assign those arguments to the variable.</p>

<pre><code>.mixin (...) { // matches 0-N arguments
.mixin () { // matches exactly 0 arguments
.mixin (@a: 1) { // matches 0-1 arguments
.mixin (@a: 1; ...) { // matches 0-N arguments
.mixin (@a; ...) { // matches 1-N arguments
</code></pre>

<p>Furthermore:</p>

<pre><code>.mixin (@a; @rest...) {
// @rest is bound to arguments after @a
// @arguments is bound to all arguments
}
</code></pre>

<h2>The Keyword !important</h2>

<p>Use the !important keyword after mixin call to mark all properties brought by it as !important:</p>

<p>Sample input:</p>

<pre><code>.mixin (@a: 0) {
border: @a;
boxer: @a;
}
.unimportant {
.mixin(1);
}
.important {
.mixin(2) !important;
}
</code></pre>

<p>compiled into:</p>

<pre><code>.unimportant {
border: 1;
boxer: 1;
}
.important {
border: 2 !important;
boxer: 2 !important;
}
</code></pre>

<h2>Pattern-matching and Guard expressions</h2>

<p>Sometimes, you may want to change the behaviour of a mixin,
based on the parameters you pass to it. Let&rsquo;s start with something
basic:</p>

<pre><code>.mixin (@s; @color) { ... }

.class {
.mixin(@switch; #888);
}
</code></pre>

<p>Now let&rsquo;s say we want <code>.mixin</code> to behave differently, based on the value of <code>@switch</code>,
we could define <code>.mixin</code> as such:</p>

<pre><code>.mixin (dark; @color) {
color: darken(@color, 10%);
}
.mixin (light; @color) {
color: lighten(@color, 10%);
}
.mixin (@_; @color) {
display: block;
}
</code></pre>

<p>Now, if we run:</p>

<pre><code>@switch: light;

.class {
.mixin(@switch; #888);
}
</code></pre>

<p>We will get the following CSS:</p>

<pre><code>.class {
color: #a2a2a2;
display: block;
}
</code></pre>

<p>Where the color passed to <code>.mixin</code> was lightened. If the value of <code>@switch</code> was <code>dark</code>,
the result would be a darker color.</p>

<p>Here&rsquo;s what happened:</p>

<ul>
<li>The first mixin definition didn&rsquo;t match because it expected <code>dark</code> as the first argument.</li>
<li>The second mixin definition matched, because it expected <code>light</code>.</li>
<li>The third mixin definition matched because it expected any value.</li>
</ul>


<p>Only mixin definitions which matched were used. Variables match and bind to any value.
Anything other than a variable matches only with a value equal to itself.</p>

<p>We can also match on arity, here&rsquo;s an example:</p>

<pre><code>.mixin (@a) {
color: @a;
}
.mixin (@a; @b) {
color: fade(@a; @b);
}
</code></pre>

<p>Now if we call <code>.mixin</code> with a single argument, we will get the output of the first definition,
but if we call it with <em>two</em> arguments, we will get the second definition, namely <code>@a</code> faded to <code>@b</code>.</p>

<h3>Guards</h3>

<p>Guards are useful when you want to match on <em>expressions</em>, as opposed to simple values or arity. If you are
familiar with functional programming, you have probably encountered them already.</p>

<p>In trying to stay as close as possible to the declarative nature of CSS, LESS has opted to implement
conditional execution via <strong>guarded mixins</strong> instead of if/else statements, in the vein of <code>@media</code>
query feature specifications.</p>

<p>Let&rsquo;s start with an example:</p>

<pre><code>.mixin (@a) when (lightness(@a) &gt;= 50%) {
background-color: black;
}
.mixin (@a) when (lightness(@a) &lt; 50%) {
background-color: white;
}
.mixin (@a) {
color: @a;
}
</code></pre>

<p>The key is the <strong><code>when</code></strong> keyword, which introduces a guard sequence (here with only one guard). Now if we run the following
code:</p>

<pre><code>.class1 { .mixin(#ddd) }
.class2 { .mixin(#555) }
</code></pre>

<p>Here&rsquo;s what we&rsquo;ll get:</p>

<pre><code>.class1 {
background-color: black;
color: #ddd;
}
.class2 {
background-color: white;
color: #555;
}
</code></pre>

<p>The full list of comparison operators usable in guards are: <strong><code>&gt; &gt;= = =&lt; &lt;</code></strong>. Additionally, the keyword <code>true</code>
is the only truthy value, making these two mixins equivalent:</p>

<pre><code>.truth (@a) when (@a) { ... }
.truth (@a) when (@a = true) { ... }
</code></pre>

<p>Any value other than the keyword <code>true</code> is falsy:</p>

<pre><code>.class {
.truth(40); // Will not match any of the above definitions.
}
</code></pre>

<p>Guards can be separated with a comma &lsquo;<code>,</code>&rsquo;&ndash;if any of the guards evaluates to true, it&rsquo;s
considered as a match:</p>

<pre><code>.mixin (@a) when (@a &gt; 10), (@a &lt; -10) { ... }
</code></pre>

<p>Note that you can also compare arguments with each other, or with non-arguments:</p>

<pre><code>@media: mobile;

.mixin (@a) when (@media = mobile) { ... }
.mixin (@a) when (@media = desktop) { ... }

.max (@a; @b) when (@a &gt; @b) { width: @a }
.max (@a; @b) when (@a &lt; @b) { width: @b }
</code></pre>

<p>Lastly, if you want to match mixins based on value type, you can use the <em>is*</em> functions:</p>

<pre><code>.mixin (@a; @b: 0) when (isnumber(@b)) { ... }
.mixin (@a; @b: black) when (iscolor(@b)) { ... }
</code></pre>

<p>Here are the basic type checking functions:</p>

<ul>
<li><code>iscolor</code></li>
<li><code>isnumber</code></li>
<li><code>isstring</code></li>
<li><code>iskeyword</code></li>
<li><code>isurl</code></li>
</ul>


<p>If you want to check if a value, in addition to being a number, is in a specific unit, you may use one of:</p>

<ul>
<li><code>ispixel</code></li>
<li><code>ispercentage</code></li>
<li><code>isem</code></li>
<li><code>isunit</code></li>
</ul>


<p>Last but not least, you may use the <strong><code>and</code></strong> keyword to provide additional conditions inside a guard:</p>

<pre><code>.mixin (@a) when (isnumber(@a)) and (@a &gt; 0) { ... }
</code></pre>

<p>And the <strong><code>not</code></strong> keyword to negate conditions:</p>

<pre><code>.mixin (@b) when not (@b &gt; 0) { ... }
</code></pre>

<h2>Nested rules</h2>

<p>LESS gives you the ability to use <em>nesting</em> instead of, or in combination with cascading.
Lets say we have the following CSS:</p>

<pre><code>#header { color: black; }
#header .navigation {
font-size: 12px;
}
#header .logo {
width: 300px;
}
#header .logo:hover {
text-decoration: none;
}
</code></pre>

<p>In LESS, we can also write it this way:</p>

<pre><code>#header {
color: black;

.navigation {
font-size: 12px;
}
.logo {
width: 300px;
&amp;:hover { text-decoration: none }
}
}
</code></pre>

<p>Or this way:</p>

<pre><code>#header { color: black;
.navigation { font-size: 12px }
.logo { width: 300px;
&amp;:hover { text-decoration: none }
}
}
</code></pre>

<p>The resulting code is more concise, and mimics the structure of your <code>DOM tree</code>.</p>

<p>Notice the <code>&amp;</code> combinator - it&rsquo;s used when you want a nested selector to be concatenated to its parent selector, instead
of acting as a descendant. This is especially important for pseudo-classes like <code>:hover</code> and <code>:focus</code>.</p>

<p>For example:</p>

<pre><code>.bordered {
&amp;.float {
float: left;
}
.top {
margin: 5px;
}
}
</code></pre>

<p>Will output</p>

<pre><code>.bordered.float {
float: left;
}
.bordered .top {
margin: 5px;
}
</code></pre>

<h2>Nested Media Queries</h2>

<p>Media queries can be nested in the same way as selectors e.g.</p>

<pre><code>.one {
@media (width: 400px) {
font-size: 1.2em;
@media print and color {
color: blue;
}
}
}
</code></pre>

<p>Will output</p>

<pre><code>@media (width: 400px) {
.one {
font-size: 1.2em;
}
}
@media (width: 400px) and print and color {
.one {
color: blue;
}
}
</code></pre>

<h2>Advanced Usage of &amp;</h2>

<p>The &amp; symbol can be used in selectors in order to reverse the ordering of the nesting and to multiply classes.</p>

<p>For example:</p>

<pre><code>.child, .sibling {
.parent &amp; {
color: black;
}
&amp; + &amp; {
color: red;
}
}
</code></pre>

<p>Will output</p>

<pre><code>.parent .child,
.parent .sibling {
color: black;
}
.child + .child,
.child + .sibling,
.sibling + .child,
.sibling + .sibling {
color: red;
}
</code></pre>

<p>You can also use &amp; in mixins in order to reference nesting that is outside of your mixin.</p>

<h2>Operations</h2>

<p>Any number, color or variable can be operated on. Operations should be performed
within parentheses. Here are a couple of examples:</p>

<pre><code>@base: 5%;
@filler: (@base * 2);
@other: (@base + @filler);

color: (#888 / 4);
background-color: (@base-color + #111);
height: (100% / 2 + @filler);
</code></pre>

<p>The output is pretty much what you expect—LESS understands the difference between colors and units. If a unit is used in an operation, like in:</p>

<pre><code>@var: (1px + 5);
</code></pre>

<p>LESS will use that unit for the final output—<code>6px</code> in this case.</p>

<p>Extra parentheses are also authorized in operations:</p>

<pre><code>width: ((@var + 5) * 2);
</code></pre>

<h2>Functions</h2>

<p>LESS provides a variety of functions which transform colors, manipulate strings and do maths.
They are documented fully in the function reference.</p>

<p>Using them is pretty straightforward. The following example uses percentage to convert 0.5 to 50%,
increases the saturation of a base color by 5% and then sets the background color to one that is lightened by
25% and spun by 8 degrees:</p>

<pre><code>@base: #f04615;
@width: 0.5;

.class {
width: percentage(0.5); // returns `50%`
color: saturate(@base, 5%);
background-color: spin(lighten(@base, 25%), 8);
}
</code></pre>

<h2>Namespaces</h2>

<p>Sometimes, you may want to group your variables or mixins, for organizational purposes, or just to offer some encapsulation.
You can do this pretty intuitively in LESS—say you want to bundle some mixins and variables under <code>#bundle</code>, for later re-use, or for distributing:</p>

<pre><code>#bundle {
.button () {
display: block;
border: 1px solid black;
background-color: grey;
&amp;:hover { background-color: white }
}
.tab { ... }
.citation { ... }
}
</code></pre>

<p>Now if we want to mixin the <code>.button</code> class in our <code>#header a</code>, we can do:</p>

<pre><code>#header a {
color: orange;
#bundle &gt; .button;
}
</code></pre>

<h2>Scope</h2>

<p>Scope in LESS is very similar to that of programming languages. Variables and mixins are first looked up locally,
and if they aren&rsquo;t found, the compiler will look in the parent scope, and so on.</p>

<pre><code>@var: red;

#page {
@var: white;
#header {
color: @var; // white
}
}

#footer {
color: @var; // red
}
</code></pre>

<h2>Comments</h2>

<p>CSS-style comments are preserved by LESS:</p>

<pre><code>/* Hello, I'm a CSS-style comment */
.class { color: black }
</code></pre>

<p>Single-line comments are also valid in LESS, but they are &lsquo;silent&rsquo;,
they don&rsquo;t show up in the compiled CSS output:</p>

<pre><code>// Hi, I'm a silent comment, I won't show up in your CSS
.class { color: white }
</code></pre>

<h2>Importing</h2>

<p>You can import both CSS and LESS files. Only LESS files import statements are processed, CSS file import statements are kept as they are. If you want to import a CSS file, and don&rsquo;t want LESS to process it, just use the <code>.css</code> extension:</p>

<pre><code>@import "lib.css";
</code></pre>

<p>Compilation makes only one change to CSS file imports: top level CSS file imports are moved on top of the sheet, right after @charset declarations.</p>

<p>Input file with import statement:</p>

<pre><code>h1 { color: green; }
@import "import/official-branding.css?urlParameter=23";
</code></pre>

<p>import statement has been moved on top:</p>

<pre><code>@import "import/official-branding.css?urlParameter=23";
h1 { color: green; }
</code></pre>

<p>Content of imported LESS file is copied into importing style sheet and compiled together with it. Importing and imported files share all mixins, namespaces, variables and other structures.</p>

<p>In addition, if the import statement has media queries specified in it, imported content is enclosed in the <code>@Media</code> declaration.</p>

<p>Imported &ldquo;library.less&rdquo;:</p>

<pre><code>@imported-color: red;
h1 { color: green; }
</code></pre>

<p>Main file imports the above library.less file:</p>

<pre><code>@import "library.less" screen and (max-width: 400px); // import with media queries
@import "library.less"; // import without media queries

.class {
color: @importedColor; // use imported variable
}
</code></pre>

<p>Compiled output:</p>

<pre><code>// Corresponds to import with media queries
@media screen and (max-width: 400px) {
h1 { color: green; }
}

// Corresponds to import without media queries
h1 { color: green; }
.class {
// Use imported variable
color: #ff0000;
}
</code></pre>

<p>LESS file import statement does not have to be located on top of the style sheet. It can be placed also inside rulesets, mixins or other LESS structures.</p>

<p>Import into ruleset:</p>

<pre><code>pre {
@import "library.less";
color: @importedColor;
}
</code></pre>

<p>both variable and ruleset defined in &ldquo;library.less&rdquo; have been copied into the <code>pre</code> ruleset:</p>

<pre><code>pre {
color: #ff0000; // variable defined in library.less was available
}
pre h1 { // ruleset defined in library.less was nested into 'pre' ruleset
color: green;
}
</code></pre>

<p>In v1.3.0 - v1.3.3 <code>@import</code> imports a file multiple times and you can override this behaviour with <code>@import-once</code>.</p>

<p>In v1.4.0 <code>@import-once</code> has been removed and <code>@import</code> imports once by default. This means that with the following</p>

<p> @import &ldquo;file.less&rdquo;;
@import &ldquo;file.less&rdquo;;</p>

<p>The second file is ignored.</p>

<p>Any file that does not end with <code>.css</code> is considered less file and processed. In addition, if the file name has no extension or parameters, the &ldquo;.less&rdquo; suffix is added on the end. Both of these are equivalent:</p>

<pre><code>@import "lib.less";
@import "lib";
</code></pre>

<p>In v1.4.0 you can force a file to be interpreted as a particular type by specifying an option, e.g.</p>

<pre><code>@import (css) "lib";
</code></pre>

<p>Will output..</p>

<pre><code>@import "lib";
</code></pre>

<p>and</p>

<pre><code>@import (less) "lib.css";
</code></pre>

<p>Will import the lib.css file and treat it as less. If you specify a file is less and do not include an extension, none will be added.</p>

<h2>String interpolation</h2>

<p>Variables can be embedded inside strings in a similar way to Ruby or PHP, with the <code>@{name}</code> construct:</p>

<pre><code>@base-url: "http://assets.fnord.com";
background-image: url("@{base-url}/images/bg.png");
</code></pre>

<h2>Escaping</h2>

<p>Sometimes you might need to output a CSS value which is either not valid CSS syntax,
or uses proprietary syntax which LESS doesn&rsquo;t recognize.</p>

<p>To output such value, we place it inside a string prefixed with <code>~</code>, for example:</p>

<pre><code>.class {
filter: ~"ms:alwaysHasItsOwnSyntax.For.Stuff()";
}
</code></pre>

<p>This is called an &ldquo;escaped value&rdquo;, which will result in:</p>

<pre><code>.class {
filter: ms:alwaysHasItsOwnSyntax.For.Stuff();
}
</code></pre>

<p>Escaped values can use the interpolation exactly the same way as strings:</p>

<pre><code>.class {
@what: "Stuff";
filter: ~"ms:alwaysHasItsOwnSyntax.For.@{what}()";
}
</code></pre>

<h2>Selector Interpolation</h2>

<p>If you want to use LESS variables inside selectors, you can do this by referencing the variable using <code>@{selector}</code> as
in string interpolation. For example:</p>

<pre><code>@name: blocked;
.@{name} {
color: black;
}
</code></pre>

<p>will output</p>

<pre><code>.blocked {
color: black;
}
</code></pre>

<p>Note: prior to LESS 1.3.1 a <code>(~"@{name}")</code> type of selector was supported. Support for this will be removed in 1.4.0.</p>

<h2>Media Queries as Variables</h2>

<p>If you want to use less variables inside media, you can do this using the usual variable variable referencing syntax <code>@variable</code>. For example:</p>

<pre><code>@singleQuery: ~"(max-width: 500px)";
@media screen, @singleQuery {
set {
padding: 3 3 3 3;
}
}
</code></pre>

<p>compiles into:</p>

<pre><code>@media screen, (max-width: 500px) {
set {
padding: 3 3 3 3;
}
}
</code></pre>

<p>The variable must contain whole media query. This would cause an error: <code>@media screen and @partial {</code>.</p>

<p>In 1.4.0, without strict maths off, you can also include variables in media values, e.g. <code>@media screen, (max-width: @width) {</code>.</p>

<h2>JavaScript evaluation</h2>

<p>JavaScript expressions can be evaluated as values inside .less files. We recommend using caution with this feature
as the LESS will not be compilable by ports and it makes the LESS harder to maintain. If possible, try to think of a
function that can be added to achieve the same purpose and ask for it on github. We have plans to allow expanding the
default functions available. However, if you still want to use JavaScript in .less, this is done by wrapping the expression
with back-ticks:</p>

<pre><code>@var: `"hello".toUpperCase() + '!'`;
</code></pre>

<p>Becomes:</p>

<pre><code>@var: "HELLO!";
</code></pre>

<p>Note that you may also use interpolation and escaping as with strings:</p>

<pre><code>@str: "hello";
@var: ~`"@{str}".toUpperCase() + '!'`;
</code></pre>

<p>Becomes:</p>

<pre><code>@var: HELLO!;
</code></pre>

<p>It is also possible to access the JavaScript environment:</p>

<pre><code>@height: `document.body.clientHeight`;
</code></pre>

<p>If you want to parse a JavaScript string as a hex color, you may use the <code>color</code> function:</p>

<pre><code>@color: color(`window.colors.baseColor`);
@darkcolor: darken(@color, 10%);
</code></pre>

<h3>escape</h3>

<p>Applies <a href="http://en.wikipedia.org/wiki/Percent-encoding">URL-encoding</a> to special characters found in the input string.</p>

<ul>
<li>Following characters are exceptions and not encoded: <code>,</code>, <code>/</code>, <code>?</code>, <code>@</code>, <code>&amp;</code>, <code>+</code>, <code>'</code>, <code>~</code>, <code>!</code> and <code>$</code>.</li>
<li>Most common encoded characters are: <code>&lt;space&gt;</code>, <code>#</code>, <code>^</code>, <code>(</code>, <code>)</code>, <code>{</code>, <code>}</code>, <code>|</code>, <code>:</code>, <code>&gt;</code>, <code>&lt;</code>, <code>;</code>, <code>]</code>, <code>[</code> and <code>=</code>.</li>
</ul>


<p>Parameters:</p>

<ul>
<li><code>string</code>: A string to escape</li>
</ul>

<pre><code>escape('a=1')

<pre><code>a%3D1
</code></pre>

<h3>e</h3>

<p>CSS escaping similar to <code>~"value"</code> syntax. It expects string as a parameter and return its content as is, but without quotes. It can be used to output CSS value which is either not valid CSS syntax, or uses proprietary syntax which LESS doesn�t recognize.</p>

<p>Parameters:</p>

<ul>
<li><code>string</code>: A string to escape</li>
</ul>


<p>Returns: <code>string</code> content without quotes.</p>

<p>Example:</p>

<h3>% format</h3>

<p>The function <code>%("format", arguments ...)</code> formats a string. The first argument is string with placeholders. All placeholders start with percentage symbol <code>%</code> followed by letter <code>s</code>,<code>S</code>,<code>d</code>,<code>D</code>,<code>a</code>, or <code>A</code>. Remaining arguments contain expressions to replace placeholders. If you need to print the percentage symbol, escape it by another percentage <code>%%</code>.</p>

<p>Use uppercase placeholders if you need to escape special characters into their utf-8 escape codes.
The function escapes all special characters except <code>()'~!</code>. Space is encoded as <code>%20</code>. Lowercase placeholders leave special characters as they are.</p>

<p>Placeholders:
* d, D, a, A - can be replaced by any kind of argument (color, number, escaped value, expression, &hellip;). If you use them in combination with string, the whole string will be used - including its quotes. However, the quotes are placed into the string as they are, they are not escaped by &ldquo;/&rdquo; nor anything similar.
* s, S - can be replaced by any kind of argument except color. If you use them in combination with string, only the string value will be used - string quotes are omitted.</p>

<p>Parameters:</p>

<ul>
<li><code>string</code>: format string with placeholders,</li>
<li><code>anything</code>* : values to replace placeholders.</li>
</ul>


<p>Returns: formatted <code>string</code>.</p>

<p>Example:</p>

<pre><code>format-a-d: %("repetitions: %a file: %d", 1 + 2, "directory/file.less");
format-a-d-upper: %('repetitions: %A file: %D', 1 + 2, "directory/file.less");
format-s: %("repetitions: %s file: %s", 1 + 2, "directory/file.less");
format-s-upper: %('repetitions: %S file: %S', 1 + 2, "directory/file.less");
</code></pre>

<p>Output:</p>

<pre><code>format-a-d: "repetitions: 3 file: "directory/file.less"";
format-a-d-upper: "repetitions: 3 file: %22directory%2Ffile.less%22";
format-s: "repetitions: 3 file: directory/file.less";
format-s-upper: "repetitions: 3 file: directory%2Ffile.less";
</code></pre>

<h3>contrast</h3>

<p>Choose which of two colors provides the greatest contrast with another. This is useful for ensuring that a color is readable against a background, which is also useful for accessibility compliance. This function works the same way as the <a href="http://compass-style.org/reference/compass/utilities/color/contrast/">contrast function in Compass for SASS</a>. In accordance with <a href="http://www.w3.org/TR/2008/REC-WCAG20-20081211/#relativeluminancedef">WCAG 2.0</a>, colors are compared using their luma value, not their lightness.</p>

<p>The light and dark parameters can be supplied in either order - the function will calculate their luma values and assign light and dark appropriately.</p>

<p>Parameters:</p>

<ul>
<li><code>color</code>: A color object to compare against.</li>
<li><code>dark</code>: optional - A designated dark color (defaults to black).</li>
<li><code>light</code>: optional - A designated light color (defaults to white).</li>
<li><code>threshold</code>: optional - A percentage 0-100% specifying where the transition from &ldquo;dark&rdquo; to &ldquo;light&rdquo; is (defaults to 43%). This is used to bias the contrast one way or another, for example to allow you to decide whether a 50% grey background should result in black or white text. You would generally set this lower for &lsquo;lighter&rsquo; palettes, higher for &lsquo;darker&rsquo; ones. Defaults to 43%.</li>
</ul>


<p>Returns: <code>color</code></p>

<p>Example:</p>

<pre><code>contrast(#aaaaaa)
contrast(#222222, #101010)
contrast(#222222, #101010, #dddddd)
contrast(hsl(90, 100%, 50%),#000000,#ffffff,40%);
contrast(hsl(90, 100%, 50%),#000000,#ffffff,60%);
</code></pre>

<p>Output:</p>

<pre><code>#000000 // black
#ffffff // white
#dddddd
#000000 // black
#ffffff // white
</code></pre>
